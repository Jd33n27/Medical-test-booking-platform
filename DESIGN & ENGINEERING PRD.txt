DESIGN & ENGINEERING PRD
Product Context
The Lab Test is a healthcare booking platform that enables users to search for laboratory tests, compare prices across accredited diagnostic centers, schedule appointments, request home sample collection, and access digital test results.
The MVP focuses on simplifying the diagnostic testing process while improving transparency and accessibility.
The product should reduce the friction associated with finding, comparing, and booking laboratory services.
Design Objective
Design an experience that helps users:
1. Immediately understand that they can compare and book laboratory tests online.
2. Quickly search and find relevant tests.
3. Compare available laboratories confidently.
4. Book appointments with minimal effort.
5. Access test results digitally.
The experience should guide users through a clear step by step journey rather than requiring extensive exploration.
Design Principles
Clarity Over Complexity
Users should immediately understand how to search, compare, and book laboratory services.
Progressive Disclosure
Present only the information necessary for the user's current task.
Trust First
Healthcare decisions require confidence. Accreditation indicators, transparent pricing, and clear information should be visible throughout the experience.
Mobile First Thinking
The platform should be designed for mobile users first and adapted for larger screens.
Accessibility
Forms, buttons, and content should remain easy to use across age groups and varying levels of technical literacy.

Content & Copy Guidance
All content should be:
* Simple
* Reassuring
* Action oriented
* Easy to understand
Avoid medical jargon where possible.
Examples:
Good:
* Book your test in minutes.
* Compare laboratories instantly.
* Download your results securely.
Avoid:
* Technical healthcare terminology.
* Long explanatory paragraphs.

Acceptance Criteria
FR1 – Search Test
Acceptance Criteria
* User can search by test name.
* Relevant tests are returned.
* Empty state appears if no result exists.
FR2 – View Laboratories
Acceptance Criteria
* User sees laboratories offering selected test.
* Prices are displayed clearly.
* Laboratory details are accessible.
FR3 – Compare Prices
Acceptance Criteria
* User can compare at least two laboratories.
* Prices are displayed consistently.
FR4 – Appointment Booking
Acceptance Criteria
* User selects available date.
* User selects available time.
* User cannot select past dates.
* Confirmation is generated after booking.
FR5 – Home Collection
Acceptance Criteria
* User submits address successfully.
* Collection request is stored.
* Confirmation is displayed.
FR6 – Results Portal
Acceptance Criteria
* User accesses uploaded results.
* User downloads result file.
* Result status is displayed.
